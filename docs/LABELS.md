# Project Labels

## Areas
- `area:frontend`
- `area:firmware`
- `area:data`
- `area:hardware`

## Types
- `type:bug`
- `type:feature`
- `type:research`

## Priority
- `priority:p0` — immediate safety, security, data-loss, or release blocker.
- `priority:p1` — highest-value active milestone work.
- `priority:p2` — planned work without immediate blocking impact.

## Workflow
- `status:blocked`
- `agent-ready`
- `needs-human-test`

Apply `agent-ready` only when the issue satisfies the definition in `docs/AGENT_OPERATIONS.md`. Apply `needs-human-test` whenever acceptance depends on visual, physical, vehicle, device, or production-environment evidence.
