# Issue Lifecycle

1. **Inbox** — idea or defect captured.
2. **Research needed** — a decision or evidence gap blocks implementation.
3. **Defined** — scope and acceptance criteria exist.
4. **Agent-ready** — dependencies, validation, and constraints are complete.
5. **In progress** — one owner and one isolated branch.
6. **Draft PR** — implementation complete enough for review.
7. **Human test** — visual, device, vehicle, or production evidence required.
8. **Ready to merge** — checks pass and review threads are resolved.
9. **Done** — merged and any required deployment or physical validation is recorded.

`agent-ready` is removed when scope changes or a new unresolved dependency appears.
